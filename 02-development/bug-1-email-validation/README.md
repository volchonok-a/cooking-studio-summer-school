# Баг 1: Отсутствие валидации email при входе

## 1. Симптом / Цель
**Симптом:** В модальном окне входа пользователь может ввести некорректный email (например, "abc" или "test@") и форма отправится без проверки. Это приводит к ошибкам на бэкенде и плохому UX.

**Цель:** Добавить клиентскую валидацию email-адреса перед отправкой формы входа.

## 2. Требования
- **FR-01:** Email должен соответствовать стандартному формату (user@domain.com)
- **UX-06:** При невалидном email пользователь должен увидеть понятное сообщение об ошибке
- **SEC-01:** Некорректные данные не должны отправляться на сервер

## 3. Реализация
Добавлена JavaScript-функция `validateEmail()`, которая:
- Проверяет email с помощью регулярного выражения
- Показывает красное сообщение "Введите корректный email" при ошибке
- Меняет цвет границы поля ввода на красный
- Блокирует отправку формы при невалидном email

## 4. Используемые инструменты
- **ИИ:** Qwen (помощь в разработке регулярного выражения)
- **Среда:** VS Code
- **Технологии:** HTML5, JavaScript, Regular Expressions

## 5. Проверка вручную
1. Открыть index.html в браузере
2. Нажать "Войти в аккаунт"
3. Ввести "abc" в поле email -> должно появиться сообщение об ошибке
4. Ввести "test@" -> должно появиться сообщение об ошибке
5. Ввести "test@example.com" -> ошибка должна исчезнуть
6. Нажать "Войти" -> форма должна отправиться
