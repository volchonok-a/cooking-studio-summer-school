# Фича 2: Валидация поля "Аллергии и особенности питания"

## 1. Симптом / Цель
**Симптом:** Поле "Аллергии и особенности питания" помечено как "Обязательно", но в HTML-форме нет реальной валидации. Пользователь может отправить форму, не заполнив это поле, что критично для безопасности клиентов с аллергиями.

**Цель:** Добавить клиентскую валидацию, которая не позволит отправить форму, если поле аллергий пустое.

## 2. Требования
- **FR-16:** Поле аллергий обязательно для заполнения
- **UX-03:** При ошибке валидации пользователь должен увидеть понятное сообщение
- **UX-04:** Кнопка "Подтвердить бронь" не должна отправлять форму при невалидных данных

## 3. Реализация
Добавлена JavaScript-функция валидации, которая:
- Проверяет заполненность поля "Аллергии" при нажатии на "Подтвердить бронь"
- Если поле пустое — показывает красное сообщение "Пожалуйста, укажите аллергии или напишите 'Нет'"
- Блокирует отправку формы до заполнения поля

## 4. Используемые инструменты
- **ИИ:** Qwen (помощь в разработке валидации)
- **Среда:** VS Code
- **Технологии:** HTML5, JavaScript, HTML5 Form Validation API

## 5. Проверка вручную
1. Открыть index.html в браузере
2. Оставить поле "Аллергии" пустым
3. Нажать "Подтвердить бронь" -> должно появиться сообщение об ошибке
4. Заполнить поле (например, "Нет") -> ошибка должна исчезнуть
5. Нажать "Подтвердить бронь" -> форма должна отправиться
