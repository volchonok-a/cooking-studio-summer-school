# Фича 1: Динамический перерасчёт стоимости в реальном времени

## 1. Симптом / Цель
**Симптом:** Пользователь не видит итоговую стоимость до нажатия кнопки "Подтвердить бронь". Это создаёт неопределённость и может привести к отказу от бронирования.

**Цель:** Реализовать автоматический перерасчёт стоимости при изменении количества мест или выборе экипировки, чтобы пользователь видел актуальную цену в реальном времени.

## 2. Требования
- **FR-13:** Количество мест влияет на итоговую стоимость
- **FR-14:** Прокат экипировки стоит 500₽ за набор
- **UX-01:** Пользователь должен видеть итог до подтверждения бронирования
- **UX-02:** Перерасчёт должен происходить мгновенно при изменении любых параметров

## 3. Реализация
Добавлен JavaScript-обработчик событий, который:
- Отслеживает изменения в поле "Количество мест"
- Отслеживает выбор радиокнопок "Экипировка студии"
- Автоматически пересчитывает стоимость по формуле: (количество мест × цена_места) + (экипировка × 500)
- Обновляет текст в блоке "Расчет стоимости..."

## 4. Используемые инструменты
- **ИИ:** Qwen (помощь в разработке JavaScript-логики)
- **Среда:** VS Code
- **Технологии:** HTML5, JavaScript (vanilla), DOM API

## 5. Проверка вручную
1. Открыть index.html в браузере
2. Изменить количество мест -> стоимость должна пересчитаться
3. Выбрать "Прокат экипировки" -> стоимость должна увеличиться на 500 × количество мест
4. Вернуться к "Своя экипировка" -> стоимость должна уменьшиться
