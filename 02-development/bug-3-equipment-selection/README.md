# Баг 3: Возможность отправить форму бронирования без выбора экипировки

## 1. Симптом / Цель
**Симптом:** Если пользователь не выберет ни один вариант экипировки (хотя по умолчанию выбрана "Своя"), форма всё равно отправится. Это может привести к некорректному расчёту стоимости.

**Цель:** Убедиться, что всегда выбран хотя бы один вариант экипировки перед отправкой формы.

## 2. Требования
- **FR-14:** Пользователь должен явно выбрать тип экипировки
- **UX-08:** По умолчанию должен быть выбран разумный вариант (своя экипировка)
- **BUS-02:** Расчёт стоимости должен быть корректным

## 3. Реализация
Добавлена проверка в функции `submitForm()`, которая:
- Проверяет, что выбран хотя бы один вариант экипировки
- Если нет — показывает сообщение об ошибке
- Блокирует отправку формы

Также добавлен атрибут `checked` к радиокнопке "Своя экипировка" для установки значения по умолчанию.

## 4. Используемые инструменты
- **ИИ:** Qwen (помощь в разработке логики)
- **Среда:** VS Code
- **Технологии:** HTML5, JavaScript

## 5. Проверка вручную
1. Открыть index.html в браузере
2. Проверить, что по умолчанию выбрана "Своя экипировка"
3. Изменить выбор на "Прокат" -> стоимость должна пересчитаться
4. Вернуться к "Своя экипировка" -> стоимость должна пересчитаться
5. Нажать "Подтвердить бронь" -> форма должна отправиться
