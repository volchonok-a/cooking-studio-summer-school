# Баг 2: Отсутствие проверки минимальной длины пароля

## 1. Симптом / Цель
**Симптом:** В модальном окне входа пользователь может ввести пароль из 1-2 символов. Это создаёт уязвимость безопасности и не соответствует лучшим практикам.

**Цель:** Добавить проверку минимальной длины пароля (не менее 6 символов).

## 2. Требования
- **SEC-02:** Пароль должен быть не менее 6 символов
- **UX-07:** При коротком пароле пользователь должен увидеть понятное сообщение
- **SEC-03:** Слабые пароли не должны приниматься системой

## 3. Реализация
Добавлена JavaScript-функция `validatePassword()`, которая:
- Проверяет длину пароля
- Показывает сообщение "Пароль должен быть не менее 6 символов" при ошибке
- Меняет цвет границы поля ввода на красный
- Блокирует отправку формы при невалидном пароле

## 4. Используемые инструменты
- **ИИ:** Qwen (помощь в разработке валидации)
- **Среда:** VS Code
- **Технологии:** HTML5, JavaScript

## 5. Проверка вручную
1. Открыть index.html в браузере
2. Нажать "Войти в аккаунт"
3. Ввести пароль "123" -> должно появиться сообщение об ошибке
4. Ввести пароль "123456" -> ошибка должна исчезнуть
5. Нажать "Войти" -> форма должна отправиться
