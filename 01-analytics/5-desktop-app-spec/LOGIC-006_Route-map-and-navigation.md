# Route Map & Navigation

## Overview
Defines the main user navigation flows and constraints[cite: 28].

## Requirements
- **Booking Path:** Must be limited to maximum 3 screens: `SCR-003` -> `SCR-004` -> `SCR-005`[cite: 22, 28].
- **Accessibility:** All navigation must support keyboard input (Tab/Enter)[cite: 22, 28].
- **Deep Linking:** Clicking a notification must redirect the user to the relevant object (e.g., specific booking)[cite: 22].
