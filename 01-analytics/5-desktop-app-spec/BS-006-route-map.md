# BS-006: Маршрутная карта (User Flow & Route Map)

**ID:** BS-006

**Тип:** Бизнес-сценарий / User Flow

**Домен:** Общий

**Приоритет:** Critical

**Статус:** Актуален

---

## Описание
Карта ключевых сценариев клиентского приложения. Связывает экраны (SCR), бизнес-правила и системные состояния. Ограничение (NFR-2): запись на класс — ≤ 3 экранов.

### Цель
Задать единый стандарт навигации и последовательности шагов для разработчиков и дизайнеров.

### Контекст использования
Используется как основа для проектирования навигации и прототипирования.

---

## Сценарии / Состояния

### Сценарий 1: Запись на класс (Happy Path)
**Маршрут:** SCR-001 → SCR-002 → SCR-003 → SCR-004 → SCR-005 → SCR-006.
**Шаги:**
1. Авторизация (Email/OAuth).
2. Выбор слота из списка (7 дней).
3. Просмотр деталей.
4. Заполнение формы (места, экипировка, аллергии обязательно).
5. Подтверждение → экран успеха.
6. Push-уведомление.

### Сценарий 2: Отмена брони
**Маршрут:** SCR-006 → SCR-007 → {Проверка времени}.
**Ветвление:**
- `≥ 12 часов`: Отмена без блокировки.
- `< 12 часов`: ⚠️ Предупреждение → Отмена + Блокировка на 7 дней.

### Сценарий 3: Оценка шефа
**Маршрут:** SCR-006 (вкладка «Прошедшие») → SCR-008.
**Шаги:** Бронь со статусом «Состоялся» → Кнопка «Оценить» → 1-5 звёзд → Отправка.

### Сценарий 4: Исключительные сценарии
- **Офлайн:** Просмотр из кэша, запрет мутаций.
- **Гонка за места:** Ошибка «Места закончились» → редирект на SCR-002.
- **Empty states:** Нет слотов / Нет результатов / Нет броней.

---

## Бизнес-правила

- **≤ 3 экрана до подтверждения** (SCR-003 → SCR-004 → SCR-005).
- **Аллергии обязательно** — кнопка неактивна без заполнения.
- **Оплата на месте** — везде подпись «Оплата на месте (наличные / перевод на карту)».
- **Desktop-first** — breakpoint ≥ 1280 px, hover + focus-состояния.
- **WCAG 2.1 AA** — контраст, клавиатурная навигация, labels.
