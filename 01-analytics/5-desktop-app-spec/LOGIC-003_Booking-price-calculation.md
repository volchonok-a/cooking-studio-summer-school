# LOGIC-003: Расчёт цены бронирования

**ID:** LOGIC-003

**Тип:** Техническая логика

**Домен:** 03. Бронирование

**Приоритет:** High

**Статус:** Актуален

---

## Описание
Алгоритм расчёта итоговой стоимости бронирования с учётом количества гостей, выбора экипировки и скидки постоянного клиента.

### Цель
Прозрачный и мгновенный пересчёт цены при изменении параметров формы.

### Контекст использования
SCR-004 (Форма бронирования), SCR-005 (Подтверждение брони).

---

## Формула расчёта
- price_final = (price_base × guest_count) + equipment_fee - discount


Где:
- `price_base` — базовая цена за 1 человека (из слота).
- `guest_count` — количество мест (1 ≤ N ≤ capacity_left).
- `equipment_fee` — доплата за экипировку:
  - `own` → 0 ₽
  - `rent` → 500 ₽ × guest_count
- `discount` — скидка постоянного клиента:
  - Если `visit_count > 5` → 5% от `(price_base × guest_count)`
  - Иначе → 0

---

## Шаги выполнения

### Шаг 0: Инициализация
- Получить `price_base` из данных слота.
- Получить `visit_count` из профиля клиента.
- Установить дефолты: `guest_count = 1`, `equipment = "own"`.

### Шаг 1: Изменение guest_count
- При изменении значения → пересчёт `price_final`.
- Валидация: `1 ≤ guest_count ≤ capacity_left`.

### Шаг 2: Переключение экипировки
- При выборе `rent` → добавить `500 × guest_count`.
- При выборе `own` → убрать доплату.

### Шаг 3: Применение скидки
- Если `visit_count > 5` → показать строку «Скидка постоянного клиента: −5%».
- Скидка применяется только к базовой стоимости, не к прокату.

### Шаг 4: Отображение в UI
- Блок «Итого» обновляется реактивно.
- Подпись «Оплата на месте (наличные / перевод на карту)» (FR-15).

---

## Пример расчёта

**Дано:**
- `price_base` = 5 000 ₽
- `guest_count` = 2
- `equipment` = rent
- `visit_count` = 7 (постоянный клиент)

**Расчёт:**
- База: 5 000 × 2 = 10 000 ₽
- Прокат: 500 × 2 = 1 000 ₽
- Скидка 5%: −500 ₽
- **Итого: 10 500 ₽**

---

## Бизнес-правила
- Скидка 5% применяется автоматически для постоянных клиентов (FR-16).
- Прокат — единый на всю бронь (нельзя для одного гостя «своя», для другого «прокат»).
- Итоговая сумма подтверждается бэкендом при создании брони.
