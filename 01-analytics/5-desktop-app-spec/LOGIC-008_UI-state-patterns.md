# UI State Patterns

## Overview
Defines standard UI behaviors for all screens in the application[cite: 25].

## States
- **Loading:** Skeleton screens prevent layout shifts[cite: 24].
- **Empty States:** Specific icons/headers/actions provided for every list section (e.g., "No slots available")[cite: 24].
- **Error States:** 
   - **Inline:** Validation errors under inputs[cite: 25].
   - **Banner:** System/Network errors on top of content[cite: 25].
   - **Modal:** Critical warnings[cite: 25].
