# LOGIC-008: Оценка шефа

**ID:** LOGIC-008

**Тип:** Техническая логика

**Домен:** 04. Управление бронированиями

**Приоритет:** Medium

**Статус:** Актуален

---

## Описание
Алгоритм выставления оценки шефу (1-5 звёзд) с проверкой статуса брони, защитой от повторной оценки и пересчётом рейтинга.

### Цель
Сбор обратной связи и формирование рейтинга шефа для других клиентов.

### Контекст использования
SCR-006 (Мои бронирования), SCR-008 (Оценка шефа).

---

## Шаги выполнения

### Шаг 0: Проверка доступности оценки
- Клиент нажимает «Оценить» в SCR-006.
- Frontend проверяет:
  - Статус брони = `completed` (состоялся).
  - Оценка ещё не поставлена.
- Если условия не выполнены → показать сообщение «Оценка доступна только для состоявшихся классов».

### Шаг 1: Выбор оценки
- Клиент выбирает 1-5 звёзд (клик или клавиатура).
- Hover: предпросмотр заполнения слева направо.
- Click: фиксация значения.
- Кнопка «Отправить оценку» становится активной.

### Шаг 2: Отправка оценки
- Запрос:

```
POST /ratings
Body: {
"booking_id": "uuid",
"chef_id": "uuid",
"stars": 5
}
```

- Успех (200):
  - Показать экран «Спасибо за оценку!».
  - Кнопка «Вернуться к бронированиям».
  - Рейтинг шефа пересчитан на бэкенде (FR-24).
- Ошибка (400):
  - «Оценка для этой брони уже выставлена» → показать поставленные звёзды в read-only.

### Шаг 3: Обновление рейтинга
- Бэкенд пересчитывает средний рейтинг шефа.
- Обновлённый рейтинг отображается в карточках слотов (SCR-003).

---

## Бизнес-правила
- Оценка доступна только для статуса `completed` (FR-23).
- Одна оценка на бронь — повторная запрещена.
- Текстовые комментарии в MVP не реализуются.
- После отправки бэкенд пересчитывает рейтинг (FR-24).
