# LOGIC-002: Фильтрация слотов

**ID:** LOGIC-002

**Тип:** Техническая логика

**Домен:** 02. Просмотр слотов

**Приоритет:** High

**Статус:** Актуален

---

## Описание
Алгоритм применения фильтров к списку слотов с AND-логикой, автоматическим обновлением и отображением активных фильтров в виде тегов.

### Цель
Быстрый поиск релевантных слотов без перезагрузки страницы.

### Контекст использования
SCR-002 (Список слотов), BS-001 (Фильтры слотов).

---

## Шаги выполнения

### Шаг 0: Инициализация фильтров
- Дефолтные значения:
  - `date_from` = сегодня
  - `date_to` = сегодня + 7 дней
  - `program` = все
  - `chef_id` = все
  - `only_available` = false

### Шаг 1: Открытие панели фильтров (BS-001)
- Клик по «Фильтры» → открытие модалки/дропдауна.
- Текущие значения подставляются из query-параметров.

### Шаг 2: Применение фильтров
1. Клиент выбирает параметры и нажимает «Применить».
2. Frontend формирует query-строку:
- GET /slots?date_from=...&date_to=...&program=...&chef_id=...&only_available=true
3. Список обновляется, URL синхронизируется.
4. Над списком появляются теги активных фильтров.

### Шаг 3: AND-логика
- Все фильтры комбинируются через **AND**: слот должен удовлетворять **всем** выбранным условиям.
- Если `only_available = true` → `capacity_left > 0`.

### Шаг 4: Сброс фильтров
- Клик по крестику на теге → удаление одного фильтра → повторный запрос.
- Клик по «Сбросить все» → возврат к дефолтным значениям.

### Шаг 5: Empty state (BS-002)
- Если результат пустой → показать «По вашему запросу ничего не найдено» + кнопка «Сбросить фильтры».

---

## Бизнес-правила
- Фильтры комбинируются по AND-логике.
- При изменении фильтров — автоматический запрос к API.
- Сброс возвращает дефолтный диапазон (7 дней).
