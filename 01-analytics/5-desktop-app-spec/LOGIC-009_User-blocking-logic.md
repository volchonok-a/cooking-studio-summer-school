# LOGIC-009: Паттерн состояний экрана (Loading / Empty / Error)

**ID:** LOGIC-009

**Тип:** Техническая логика / UX-паттерн

**Домен:** Общий (UX)

**Приоритет:** High

**Статус:** Актуален

---

## Описание
Единый паттерн обработки трёх базовых состояний любого экрана с данными: загрузка, пустое состояние, ошибка.

### Цель
Предсказуемое поведение интерфейса во всех ситуациях.

### Контекст использования
Все экраны: SCR-002, SCR-003, SCR-006, SCR-009, SCR-010.

---

## Состояния

### Состояние 1: Loading (загрузка)
- **Когда:** первичная загрузка данных.
- **UI:** skeleton-экран или spinner.
- **Длительность:** p95 < 2.0 с (NFR-4).
- **Применение:** SCR-002, SCR-003, SCR-006, SCR-009, SCR-010.

### Состояние 2: Empty (нет данных)
- **Когда:** данные загружены, но список пуст.
- **UI:** иконка + заголовок + текст + кнопка действия (если уместно).
- **Применение:** см. BS-002.

### Состояние 3: Error (ошибка)
- **Когда:** запрос завершился с ошибкой (сеть, сервер, валидация).
- **UI:** зависит от типа ошибки (см. BS-003):
  - Inline — под полем.
  - Banner — сверху экрана.
  - Toast — в углу.
  - Modal — центрированное окно.
  - Empty/Error — полный экран.

### Состояние 4: Offline (офлайн)
- **Когда:** `navigator.onLine === false`.
- **UI:** баннер «Нет подключения» + данные из кэша с плашкой «Данные могли устареть».
- **Применение:** см. LOGIC-006.

---

## Бизнес-правила
- При первичной загрузке — skeleton.
- При ошибке на уже загруженной странице — не ломать вёрстку, показать баннер.
- Все тексты ошибок — дружелюбные, не технические (BS-003).
