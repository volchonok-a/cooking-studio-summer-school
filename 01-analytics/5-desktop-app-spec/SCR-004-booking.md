# SCR-004: Форма бронирования

## 1. Описание и назначение
Экран служит для оформления заявки на бронирование слота. Пользователь выбирает количество мест, наличие экипировки и указывает особенности питания.

**Связи:** FR-11, FR-12, FR-14, US-6, NFR-2

---

## 2. Элементы интерфейса и поведение
### 2.1. Поля ввода
- **Количество мест (Number Input):**
  - Диапазон: 1 – `available_places`.
  - По умолчанию: 1.
  - Поведение: при изменении значения сумма в блоке «Итого» пересчитывается с debounce 200мс.
- **Экипировка (Radio Group):**
  - Опции: «Своя» (default), «Прокат» (+500 ₽).
  - Смена выбора мгновенно обновляет итоговую стоимость.
- **Аллергии (Textarea):**
  - Ограничение: 500 символов.
  - Валидация: Required. Поле считается заполненным, если введено > 3 символов (предотвращение ввода пробелов).

### 2.2. Блок «Итого»
- Отображает детализацию: `(Price * Places) + RentalFee - Discount`.
- При `LoyaltyStatus = true` скидка отображается отдельной строкой красным цветом.

---

## 3. Бизнес-правила и валидация
- **Блокировка:** Если пользователь заблокирован (`is_blocked: true`), кнопка «Подтвердить» скрывается, вместо нее выводится баннер с датой окончания блокировки.
- **Конкурентность:** При попытке бронирования, если места закончились за время заполнения формы, сервер возвращает `409 Conflict`. UI должен показать модальное окно: «К сожалению, места закончились».
- **Idempotency:** Запрос отправляется с заголовком `Idempotency-Key` для предотвращения дублей при случайном нажатии.

---

## 4. Состояния (States)
- **Loading:** Skeleton для полей.
- **Success:** Переход на `SCR-005`.
- **Error:** - Inline-валидация для поля «Аллергии».
  - Banner-ошибка для серверных сбоев.
