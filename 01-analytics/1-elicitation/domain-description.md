# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений.
> Заполняется после ответов заказчика (customer-questions.md).

## Описание домена

**Бизнес.** «Шеф-стол» — кулинарная студия в лофте на бывшем заводе, работающая третий сезон. Студия организует **групповые кулинарные классы** под руководством шеф-повара. Класс длится около 3 часов и проходит по одной из **программ**: простая итальянская кухня для новичков, сложная азиатская для опытных и т.д. Каждый класс ведёт **шеф** (сейчас их пятеро, в сезон привлекаются приглашённые).

**Процесс.** Владелец заранее формирует **расписание на неделю вперёд**: для каждого класса задаются дата и время старта, программа, назначенный шеф и число мест. В студии **12 рабочих столов**, но на классы с духовкой и сложной техникой берётся **не более 8 человек** (иначе не хватает плит и шеф не успевает за всеми). Клиент видит свободные **слоты** и **записывается**, указывая, со своим ли он фартуком и ножами или возьмёт **напрокат**. До класса клиент может **отменить** запись — при ранней отмене место освобождается, поздняя отмена приводит к порче закупленных продуктов. Сейчас запись клиентов ведётся вручную (WhatsApp + Google-таблица), что приводит к двойным записям и путанице, особенно в выходные.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на кулинарные классы (просмотр и фильтрация слотов, бронирование, выбор экипировки, отмена, свои бронирования), исключив ручную путаницу и двойные брони. Управление расписанием и работу шефов/владельца обеспечивает **существующая инфраструктура**; приложение потребляет её данные через API.

**Ключевые объекты домена:**

- **Класс / слот** — конкретное занятие: дата/время старта, программа, шеф, всего и свободно мест. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Программа** — вариант класса (итальянская, азиатская и т.д.), определяет меню, длительность и потолок мест (справочные данные, read-only).
- **Шеф** — ведёт класс; для клиента — справочные данные слота (read-only). После класса клиент может поставить шефу **оценку** (1–5 звёзд).
- **Экипировка (фартук, ножи)** — инвентарь студии; может выдаваться напрокат за дополнительную плату.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: число мест, своя/прокатная экипировка, аллергии, статус.
- **Клиент** — записывается на классы, отменяет запись, оценивает шефа. Постоянные клиенты (>5 посещений) получают скидку 5%.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское веб-приложение и API для него.**
> Роли Шефа и Владельца/Администратора обслуживаются **существующей инфраструктурой** (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость студии, записывается на кулинарные классы. Пользуется с телефона (веб-приложение, mobile-first). | Увидеть и отфильтровать свободные слоты, записаться (себя и гостей), выбрать свою/прокатную экипировку, указать аллергии, видеть свои бронирования, отменить запись, оценить шефа, получать напоминания. | **Да** |

> Шеф и Владелец/Администратор взаимодействуют с системой через **уже реализованные** интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные ими данные (слоты, программы, шефы) через API.

## Ограничения

### Бизнес-ограничения

- Всего **12 рабочих столов** — общий потолок.
- Группа на класс с духовкой/сложной техникой — **не более 8 человек**; обычный класс — до 12.
- Лимит мест слота определяется программой (требует ли она духовку).
- **Политика отмен:** бесплатная отмена не позднее чем за **12 часов** до старта; поздняя отмена (<12 ч) или неявка (no-show) приводит к **блокировке клиента на 7 дней**.
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и показывает цену.
- **Прокат экипировки** (фартук, ножи) — за дополнительную плату, оплата на месте. Лимита на количество прокатных наборов нет.
- **Поле «Аллергии и особенности питания»** — обязательное для заполнения при бронировании.
- **Программа лояльности:** клиент с **>5 посещениями** считается постоянным и получает **скидку 5%**.
- **Период списка слотов по умолчанию — 7 дней (R-027).** По умолчанию клиенту показываются слоты на ближайшие 7 дней от текущего момента. Более длинный период доступен через фильтр дат. При отсутствии расписания показывается empty state («Пока нет доступных классов»).
- **Отмена класса студией / по форс-мажору (R-008).** Если класс отменяет студия (организатор или из-за срыва поставки / поломки оборудования), бронь на стороне клиента **не удаляется**, а получает статус **«Отменён студией»** с указанием причины; клиенту приходит push-уведомление. Повторная запись на отменённый слот запрещена.

### Технические / платформенные

- **Web, mobile-first:** клиентское веб-приложение рассчитано на использование с телефона (адаптивная вёрстка).
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, программы и шефы доступны только для чтения.
- **Граница серверной интеграции (R-004).** Детали серверной реализации — транзакционность, SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**. Существующий бэкенд — **black-box источник истины**: гарантия «**0 двойных броней**» обеспечивается на его стороне. Со стороны клиента граница закрывается **контрактом API**.
- **Данные существующей инфраструктуры (R-015).** Это учебный/тестовый проект, исторических данных и легаси-хемы нет. Канонической схемой считается контракт API.
- **Авторизация:** Email + пароль или OAuth (Google/Яндекс). SMS-авторизация исключена для упрощения MVP.
- **Уведомления:** Web Push API + внутриприложенные уведомления (иконка колокольчика).

### Проектные

- **Срок** — к началу нового сезона, ~2 месяца (жёсткая цель).
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» позже.

### Границы скоупа

Скоуп — **клиентское веб-приложение и API для него**. Управление расписанием, роли шефа и владельца, админка — это **существующая инфраструктура**, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента (Email + пароль / OAuth) | Роли Шефа и Владельца/Администратора, админ-экран |
| Просмотр и **фильтрация** списка слотов (дата, программа, наличие мест, шеф) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись, в т.ч. несколько мест за раз | Онлайн-оплата |
| Выбор «своя / прокатная экипировка», обязательное поле аллергий | Программа лояльности со сложной механикой (в MVP — только счётчик + скидка 5%) |
| Список своих бронирований | Автоматический учёт инвентаря прокатной экипировки |
| Отмена записи с правилом 12 часов + блокировка при нарушении | Отметка явки/неявки шефом (выполняется в существующей инфраструктуре) |
| Оценка шефа (1–5 звёзд) после посещения | Текстовые комментарии к оценке |
| Напоминания о записи (Web Push + in-app) | SMS / Email-уведомления |
| Отображение скидки 5% для постоянных клиентов (>5 посещений) | Сложные механики бонусов и реферальных программ |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Класс / слот** | Запланированное групповое занятие: дата/время старта, программа, шеф, всего и свободно мест. |
| **Программа** | Вариант кулинарного класса (итальянская, азиатская и т.д.), определяет меню, длительность и лимит мест. |
| **Шеф** | Повар, ведущий класс; оценивается клиентами по 5-звёздочной шкале. |
| **Экипировка** | Фартук и набор ножей; могут быть своими или выдаваться студией напрокат за доп. плату. |
| **Прокат** | Выдача студийной экипировки клиенту, у которого нет своей. |
| **Своя экипировка** | Клиент приходит со своим фартуком и ножами; занимает место, но не использует прокатный фонд. |
| **Запись (бронь)** | Резерв одного или нескольких мест на конкретный слот: число мест, своя/прокатная экипировка, аллергии, статус. |
| **Отмена** | Снятие записи. Клиентом: ранняя (≥12 ч до старта) освобождает место; поздняя (<12 ч) приводит к блокировке клиента на 7 дней. Студией: бронь переходит в статус «Отменён студией» с причиной, клиент получает push, повторная запись запрещена. |
| **Отмена студией** | Отмена класса организатором или по форс-мажору (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменён студией» + причина; отправляется push-уведомление. |
| **No-show** | Клиент не пришёл на класс без отмены. Приравнивается к поздней отмене: блокировка на 7 дней. |
| **Постоянный клиент** | Клиент с более чем 5 посещениями; получает скидку 5%. |
| **Аллергии** | Обязательное поле при бронировании: информация об аллергиях и особенностях питания клиента. |
| **Владелец / администратор** | Артём — управляет расписанием и шефами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту сезона. |
