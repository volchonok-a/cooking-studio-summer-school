# Доменная модель — Кулинарная студия «Шеф-стол»

**Дата:** 05.07.2026  
**Версия:** 1.1 (Адаптивное веб-приложение, MVP)  
**Платформа:** Адаптивное веб-приложение (Mobile-first)

## Описание

Доменная модель описывает основные бизнес-сущности системы бронирования кулинарных классов и связи между ними. Модель не зависит от платформы (мобильная/веб), но атрибуты и правила учтены под специфику веб-приложения (авторизация, уведомления).

## Сущности

### 1. Client (Клиент)
Пользователь веб-приложения, который записывается на кулинарные классы.

**Атрибуты:**
- `id` — уникальный идентификатор
- `email` — email (основной способ авторизации)
- `passwordHash` — хэш пароля (или ссылка на OAuth-провайдер)
- `name` — имя
- `phone` — номер телефона (опционально, для экстренной связи)
- `allergies` — аллергии и особенности питания (обязательное поле при бронировании)
- `visitCount` — количество посещённых классов
- `isLoyal` — флаг постоянного клиента (visitCount > 5)
- `discountPercent` — процент скидки (5% для постоянных клиентов)
- `isBlocked` — флаг блокировки (при no-show или поздней отмене)
- `blockUntil` — дата окончания блокировки

**Бизнес-правила:**
- Если `visitCount > 5`, клиент считается постоянным и получает скидку 5%.
- При отмене за <12 часов или no-show клиент блокируется на 7 дней.
- Авторизация осуществляется по Email/Паролю или через соц. сети (Google/Яндекс).

---

### 2. Slot (Слот/Класс)
Конкретное занятие на определённую дату и время.

**Атрибуты:**
- `id` — уникальный идентификатор
- `programId` — ссылка на программу
- `chefId` — ссылка на шефа
- `startTime` — дата и время начала
- `duration` — длительность (по умолчанию 3 часа)
- `maxCapacity` — максимальное количество мест (8 или 12 в зависимости от программы)
- `availableSeats` — количество свободных мест
- `price` — стоимость участия
- `equipmentRentalPrice` — стоимость проката экипировки
- `status` — статус слота (active, cancelled, completed)
- `cancellationReason` — причина отмены (если status = cancelled)

**Бизнес-правила:**
- По умолчанию показываются слоты на ближайшие 7 дней.
- Если `program.requiresOven = true`, то `maxCapacity = 8`, иначе `maxCapacity = 12`.
- При отмене студией статус меняется на `cancelled`, повторная запись запрещена.

---

### 3. Program (Программа)
Тип кулинарного класса (итальянская кухня, азиатская и т.д.).

**Атрибуты:**
- `id` — уникальный идентификатор
- `name` — название (например, "Итальянская паста для новичков")
- `description` — описание
- `difficulty` — уровень сложности (beginner, intermediate, advanced)
- `menu` — меню (список блюд)
- `requiresOven` — требует ли духовку (влияет на maxCapacity)
- `duration` — стандартная длительность

---

### 4. Chef (Шеф)
Ведущий кулинарного класса.

**Атрибуты:**
- `id` — уникальный идентификатор
- `name` — имя
- `photo` — фото (для отображения в веб-интерфейсе)
- `specialization` — специализация (итальянская, азиатская и т.д.)
- `averageRating` — средний рейтинг
- `ratingCount` — количество оценок

**Бизнес-правила:**
- Рейтинг рассчитывается как среднее всех оценок от клиентов.

---

### 5. Booking (Бронь)
Запись клиента на конкретный слот.

**Атрибуты:**
- `id` — уникальный идентификатор
- `clientId` — ссылка на клиента
- `slotId` — ссылка на слот
- `seatsCount` — количество забронированных мест
- `equipmentRental` — флаг проката экипировки (true/false)
- `status` — статус брони (confirmed, cancelled_by_client, cancelled_by_studio, completed, no_show)
- `createdAt` — дата создания брони
- `cancelledAt` — дата отмены (если отменена)

**Бизнес-правила:**
- Бесплатная отмена возможна за ≥12 часов до начала.
- При отмене за <12 часов клиент блокируется на 7 дней.
- При no-show клиент блокируется на 7 дней.
- При отмене студией статус меняется на `cancelled_by_studio`, клиент не блокируется.

---

### 6. EquipmentRental (Прокат экипировки)
Дополнительная услуга — прокат фартука и набора ножей.

**Атрибуты:**
- `bookingId` — ссылка на бронь
- `apron` — флаг проката фартука (true/false)
- `knives` — флаг проката ножей (true/false)
- `price` — стоимость проката (берётся из Slot.equipmentRentalPrice)

**Бизнес-правила:**
- Лимита на количество прокатных наборов нет (для MVP).
- Оплата за прокат производится на месте.

---

### 7. Rating (Оценка)
Оценка шефа клиентом после посещения класса.

**Атрибуты:**
- `id` — уникальный идентификатор
- `bookingId` — ссылка на бронь
- `chefId` — ссылка на шефа
- `clientId` — ссылка на клиента
- `stars` — оценка (1-5 звёзд)
- `createdAt` — дата оценки

**Бизнес-правила:**
- Оценка ставится после завершения класса (status = completed).
- Один клиент может оценить шефа только один раз за посещение.

---

### 8. Notification (Уведомление)
Web Push-уведомление или внутриприложенное уведомление (колокольчик/toast) для клиента.

**Атрибуты:**
- `id` — уникальный идентификатор
- `clientId` — ссылка на клиента
- `type` — тип уведомления (booking_confirmed, reminder_1day, studio_cancelled)
- `title` — заголовок
- `message` — текст уведомления
- `sentAt` — дата отправки
- `readAt` — дата прочтения (опционально)
- `channel` — канал доставки (web_push, in_app)

**Типы уведомлений (MVP):**
- `booking_confirmed` — подтверждение брони.
- `reminder_1day` — напоминание за день до класса.
- `studio_cancelled` — отмена класса студией.

---

## Связи между сущностями

```text
Client 1───* Booking *───1 Slot
                │              │
                │              ├──1 Program
                │              └──1 Chef
                │
                ├──1 EquipmentRental
                │
                └──* Rating ──1 Chef

Client 1───* Notification

```

### Описание связей

| Связь | Кардинальность | Описание |
|---|---|---|
| **Client → Booking** | 1 : * | Один клиент может сделать много броней |
| **Booking → Slot** | * : 1 | Одна бронь относится к одному слоту |
| **Slot → Program** | 1 : 1 | Один слот имеет одну программу |
| **Slot → Chef** | 1 : 1 | Один слот ведёт один шеф |
| **Booking → EquipmentRental** | 1 : 1 | Одна бронь может иметь один прокат экипировки |
| **Booking → Rating** | 1 : 1 | После завершения брони клиент может поставить одну оценку шефу |
| **Client → Notification** | 1 : * | Один клиент получает много уведомлений |
