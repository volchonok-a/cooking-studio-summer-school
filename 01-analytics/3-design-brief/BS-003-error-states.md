# BS-003: Паттерны состояний ошибок (Error States)

## 1. Описание
Документ описывает единые паттерны отображения ошибок в клиентском приложении. Все ошибки делятся на системные (сеть, таймауты), валидации и бизнес-логики. Цель — сделать сообщения понятными, не техническими и предлагать действие для исправления.

---

## 2. Глобальные паттерны отображения ошибок

| Тип ошибки | UI-элемент | Где применяется |
|---|---|---|
| **Inline (под полем)** | Красный текст под инпутом | Ошибки валидации форм (например, пустые аллергии) |
| **Banner (плашка сверху)** | Жёлтая/красная плашка на всю ширину контента | Ошибки загрузки списка, офлайн-режим, блокировка аккаунта |
| **Toast (всплывашка)** | Небольшое уведомление в углу экрана | Успешные действия, короткие системные ошибки (редко) |
| **Modal (модалка)** | Центрированное окно с затемнением фона | Критические предупреждения (поздняя отмена), подтверждение действий |
| **Empty/Error State** | Иконка + текст + кнопка действия | Полный экран при ошибке загрузки или отсутствии данных |

---

## 3. Сценарии ошибок

### 3.1. Ошибки авторизации (UC-1)

**Неверный Email или пароль**
- **Тип:** Inline
- **Текст:** «Неверный Email или пароль»
- **Поведение:** Поля очищаются, фокус устанавливается на поле Email.

**OAuth-провайдер недоступен (NFR-10)**
- **Тип:** Inline / Toast
- **Текст:** «Сервис временно недоступен, попробуйте Email + пароль»
- **Поведение:** Кнопка OAuth становится `disabled`.

---

### 3.2. Ошибки валидации (UC-6)

**Пустое поле «Аллергии» (FR-12)**
- **Тип:** Inline
- **Текст:** «Укажите аллергии или "Нет / не беспокоят"»
- **Поведение:** Кнопка «Подтвердить бронь» остаётся `disabled`. Поле подсвечивается красным при попытке отправить форму.

**Некорректный Email**
- **Тип:** Inline
- **Текст:** «Введите корректный Email»
- **Поведение:** Кнопка «Войти» `disabled`.

---

### 3.3. Ошибки бизнес-логики

**Нет мест в слоте / Гонка за места (NFR-7, UC-6 E2)**
- **Тип:** Modal / Error State
- **Заголовок:** «Места закончились»
- **Текст:** «К сожалению, пока вы заполняли форму, места заняли. Выберите другой слот.»
- **Действие:** Кнопка «Вернуться к списку» (редирект на SCR-002).

**Клиент заблокирован (FR-29, UC-6 E3)**
- **Тип:** Banner
- **Текст:** «Записи недоступны до <дата>»
- **Подтекст:** «Блокировка наложена из-за поздней отмены или неявки.»
- **Поведение:** Все кнопки «Записаться» на странице `disabled`.

**Слот отменён студией (FR-22)**
- **Тип:** Banner / Status Badge
- **Текст:** «Класс отменён студией. Причина: <причина>»
- **Поведение:** Кнопка «Записаться» `disabled`.

---

### 3.4. Сетевые ошибки и таймауты (NFR-9)

**Таймаут запроса (~10 секунд)**
- **Тип:** Banner / Error State
- **Текст:** «Превышено время ожидания ответа сервера»
- **Действие:** Кнопка «Повторить».

**Потеря соединения (Offline)**
- **Тип:** Banner (постоянная плашка сверху)
- **Текст:** «Нет подключения к интернету»
- **Поведение:** Запрет мутаций (кнопки действий `disabled`). Просмотр кэшированных данных с плашкой «Данные могли устареть».

**Неопределённый результат создания брони (Таймаут при `createBooking`)**
- **Тип:** Modal
- **Текст:** «Не удалось получить подтверждение. Возможно, бронь уже создана.»
- **Действие:** Кнопка «Проверить мои бронирования» (переход на SCR-006 с использованием `Idempotency-Key` для безопасного повтора).

---

## 4. Копирайт ошибок (Тон коммуникации)

1. **Не обвинять пользователя.** Вместо «Вы не заполнили поле» → «Пожалуйста, укажите аллергии».
2. **Не показывать технические коды.** Вместо «Error 500: Internal Server Error» → «Что-то пошло не так. Попробуйте ещё раз».
3. **Предлагать решение.** Каждая ошибка должна заканчиваться понятным действием (кнопка «Повторить», «Сбросить фильтры», «Вернуться к списку»).
4. **Язык:** Русский (NFR-20).
