# BS-006: Маршрутная карта (User Flow & Route Map)

## 1. Описание
Карта сценариев клиентского приложения. Связывает экраны (SCR), бизнес-правила и системные состояния.

**Ограничение (NFR-2):** запись на класс — **≤ 3 экранов** (SCR-003 → SCR-004 → SCR-005).

---

## 2. Экраны (SCR)

| ID | Название | Описание |
|---|---|---|
| SCR-001 | Авторизация | Email+пароль / OAuth |
| SCR-002 | Список слотов | 7 дней, фильтры |
| SCR-003 | Детали слота | Информация + «Записаться» |
| SCR-004 | Форма брони | Места, экипировка, аллергии |
| SCR-005 | Подтверждение | Успех, статус «Ожидает оплаты» |
| SCR-006 | Мои бронирования | Предстоящие / Прошедшие / Отменённые |
| SCR-007 | Отмена | Подтверждение с предупреждением |
| SCR-008 | Оценка шефа | 1-5 звёзд |
| SCR-009 | Профиль | Счётчик, бейдж, блокировка |
| SCR-010 | Уведомления | Колокольчик |

## 3. Основные сценарии

### 3.1. Запись на класс (Happy Path)
SCR-001 → SCR-002 → SCR-003 → SCR-004 → SCR-005 → SCR-006
(Вход)    (Слоты)   (Детали)  (Бронь)   (Успех)   (Список)


**Шаги:**
1. Авторизация (Email/OAuth)
2. Выбор слота из списка (7 дней)
3. Просмотр деталей
4. Заполнение формы (места, экипировка, **аллергии обязательно**)
5. Подтверждение → экран успеха
6. Push-уведомление

---

### 3.2. Отмена брони
SCR-006 → SCR-007 → {Проверка времени}
↓
┌─────┴─────┐
↓ ↓
≥ 12 часов < 12 часов
↓ ↓
Отмена ⚠️ Предупреждение:
без «Блокировка 7 дней»
блокировки ↓
{Подтверждение}
↓
Отмена + Блокировка


**Правила:**
- ≥12 часов: место освобождается, без блокировки
- <12 часов: блокировка на 7 дней (FR-20, FR-29)

---

### 3.3. Блокировка клиента
Попытка записаться во время блокировки
↓
Баннер: «Записи недоступны до <дата>»
↓
Все кнопки «Записаться» → disabled
↓
Через 7 дней → автоматическая разблокировка


**Причины:** поздняя отмена (<12 ч) или no-show

---

### 3.4. Оценка шефа
SCR-006 (вкладка «Прошедшие»)
↓
Бронь со статусом «Состоялся»
↓
Кнопка «Оценить» → SCR-008
↓
1-5 звёзд → Отправка
↓
Рейтинг шефа обновлён

**Правила:** одна оценка на бронь, без текста (FR-23)

---

### 3.5. Уведомления

**Push (Web Push API):**
- Напоминание за ~24 часа до старта
- Отмена студией (с причиной)
- Блокировка / разблокировка

**In-app (колокольчик SCR-010):**
- Подтверждение брони
- Отмена клиентом
- Отмена студией
- Окончание блокировки

## 4. Исключительные сценарии

### 4.1. Офлайн-режим (NFR-9)
- Просмотр: данные из кэша с плашкой «Данные могли устареть»
- Мутации: запрещены, сообщение «Нет подключения»

### 4.2. Гонка за места (NFR-7)
- Если места заняты во время бронирования → ошибка «Места закончились»
- Гарантия: 0 двойных броней (атомарность на бэкенде)

### 4.3. Empty states
- Нет слотов: «Пока нет доступных классов» + кнопка «Расширить диапазон»
- Нет результатов фильтра: «По вашему запросу ничего не найдено»
- Нет броней: «У вас пока нет бронирований»

### 4.4. Ошибки авторизации
- Неверный Email/пароль: «Неверный Email или пароль»
- OAuth недоступен: fallback на Email+пароль (NFR-10)

---

## 5. Ключевые UX-правила

1. **≤ 3 экрана до подтверждения** (SCR-003 → SCR-004 → SCR-005)
2. **Аллергии обязательно** — кнопка неактивна без заполнения (FR-12)
3. **Оплата на месте** — везде подпись «Оплата на месте (наличные / перевод на карту)»
4. **Desktop-first** — breakpoint ≥ 1280 px, hover + focus-состояния (NFR-1)
5. **WCAG 2.1 AA** — контраст, клавиатурная навигация, labels (NFR-19)
