# Дизайн-бриф: клиентское веб-приложение кулинарной студии

## 1. О проекте

**Продукт:** клиентское веб-приложение для самостоятельной записи гостей на кулинарные классы  
**Заказчик:** владелец кулинарной студии (лофт на бывшем заводе)  
**Проблема:** сейчас запись ведётся через WhatsApp и Google-таблицы — это вызывает двойные брони, путаницу, простой мест  
**Цель MVP:** запустить продукт к началу сезона (~2 месяца), убрать хаос записи, дать клиентам самостоятельную запись

## 2. Целевая аудитория

**Роль:** Клиент (гость студии)  
**Портрет:**
- записывается на кулинарный класс (часто компанией);
- пользуется приложением с компьютера (дом, офис);
- хочет быстро выбрать время и записаться без звонков.

## 4. Что НЕ проектируем (границы MVP)

- Онлайн-оплату и экраны чеков/карт (только подпись «оплата на месте»)
- SMS/Email-уведомления (только Web Push + in-app)
- Интерфейсы шефа и администратора
- Сложные механики лояльности (только счётчик + бейдж + скидка 5%)
- Мобильную версию как приоритет
