# SCR-006: Мои бронирования

## Описание
Клиент видит список своих бронирований с фильтрами по статусу.

## Элементы интерфейса

### Вкладки/фильтры
- **Предстоящие** (по умолчанию)
- **Прошедшие**
- **Отменённые**

### Карточка брони
Для каждой брони:
- **Дата и время**
- **Название программы**
- **Шеф** (имя)
- **Статус** (бейдж):
  - «Ожидает оплаты» (серый)
  - «Состоялся» (зелёный)
  - «Отменено клиентом» (жёлтый)
  - «Отменён студией» (красный)
- **Количество мест**
- **Экипировка**
- **Аллергии** (кратко)

### Кнопки действия

#### Для предстоящих:
- **«Отменить»** (ведёт на SCR-007)

#### Для прошедших (статус «Состоялся»):
- **«Оценить»** (ведёт на SCR-008, если ещё не оценено)
- Или текст: **«Вы уже оценили»**

#### Для отменённых:
- Действий нет

## Состояния

### 1. Empty state (нет броней)
- Текст: **«У вас пока нет бронирований»**
- Подтекст: **«Запишитесь на первый класс!»**
- Кнопка **«Перейти к классам»**

### 2. Empty state (нет в категории)
- Например, в «Прошедших»: **«У вас пока нет прошедших классов»**

### 3. Офлайн
- Данные из кэша с плашкой «Данные могли устареть»

## Бизнес-правила
- Оценка доступна только один раз на бронь
- Отмена возможна только для предстоящих броней
