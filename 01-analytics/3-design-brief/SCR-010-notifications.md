# SCR-010: Уведомления (колокольчик)

## 1. Описание
In-app уведомления в виде выпадающего списка по клику на иконку колокольчика в шапке приложения. Бейдж непрочитанных отображается на иконке. Только Web Push API + in-app (SMS/Email вне MVP).

**Связи:** UC-10, FR-25, FR-26, FR-27, FR-28, US-13, US-15, US-16

---

## 2. Элементы интерфейса

### Иконка колокольчика (в шапке)
- Расположение: правая часть шапки, рядом с профилем
- Бейдж непрочитанных: красная точка/число (например, «3»)
- Hover: tooltip «Уведомления»
- Click: открывает выпадающий список

### Выпадающий список уведомлений
- Заголовок: **«Уведомления»**
- Ссылка **«Отметить все как прочитанные»** (если есть непрочитанные)
- Список событий (сортировка: новые сверху)
- Empty state: **«Нет новых уведомлений»** (если список пуст)

### Карточка уведомления
- Иконка типа события (бронь, отмена, блокировка)
- Заголовок события (кратко)
- Текст события (1–2 строки)
- Время («2 часа назад», «Вчера»)
- Статус: непрочитанное (жирный текст / фон) / прочитанное (обычный)
- Click: переход к связанному экрану (бронь, слот и т.д.)

---

## 3. Типы уведомлений

| Событие | Заголовок | Текст | Push | Приоритет |
|---|---|---|---|---|
| Подтверждение брони | «Бронь подтверждена» | «Итальянская кухня, 10 июля, 15:00» | ✅ | обычный |
| Напоминание (~24 ч) | «Напоминание о классе» | «Завтра в 15:00 класс „Итальянская кухня". Ждём вас!» | ✅ | обычный |
| Отмена студией | «Класс отменён» | „Итальянская кухня" 10 июля отменён. Причина: болезнь шефа.» | ✅ | важный |
| Отмена клиентом | «Вы отменили бронь» | «Бронь на 10 июля отменена» | — | обычный |
| Блокировка | «Аккаунт заблокирован» | «Записи недоступны до 15 июля из-за поздней отмены» | ✅ | важный |
| Окончание блокировки | «Блокировка снята» | «Теперь вы можете записываться на классы!» | ✅ | обычный |

## 4. Состояния

### 4.1. Есть непрочитанные уведомления
- Бейдж на колокольчике: число непрочитанных (например, «3»)
- Непрочитанные в списке выделены (жирный текст / фон)
- Ссылка «Отметить все как прочитанные» видна

### 4.2. Все уведомления прочитаны
- Бейдж на колокольчике скрыт
- Все уведомления в списке в обычном состоянии
- Ссылка «Отметить все как прочитанные» скрыта

### 4.3. Empty state (нет уведомлений)
- Иконка колокольчика (серая)
- Текст: **«Нет новых уведомлений»**
- Подтекст: «Здесь будут появляться важные события»

### 4.4. Загрузка списка
- Skeleton для 3–4 карточек
- Длительность: p95 < 2.0 с (NFR-4)

### 4.5. Ошибка загрузки
- Сообщение: **«Не удалось загрузить уведомления»**
- Кнопка **«Повторить»**

### 4.6. Запрос разрешения на Push (FR-27)
- Модалка/баннер при первом входе или первой брони
- Заголовок: **«Разрешить уведомления?»**
- Текст: «Мы будем напоминать о классах и важных изменениях»
- Кнопки: **«Разрешить»** / **«Не сейчас»**
- При отказе — уведомления работают только in-app (колокольчик)

---

## 5. Бизнес-правила

- Только Web Push API + in-app (колокольчик). SMS/Email — вне MVP (NFR-16)
- Push-токен регистрируется на бэкенде после получения системного разрешения (NFR-16)
- При отзыве разрешения на push — отправка прекращается без ошибки для пользователя
- Напоминание о классе отправляется за ~24 часа до старта (FR-25)
- При отмене класса студией — push с причиной (FR-26)
- Бейдж непрочитанных уменьшается при прочтении (FR-28)
- Клик по уведомлению ведёт на связанный экран (бронь, слот, профиль)

---

## 6. Доступность (NFR-19)

- Иконка колокольчика имеет label «Уведомления» для screen reader
- Бейдж непрочитанных озвучивается («3 непрочитанных уведомления»)
- Выпадающий список доступен через клавиатуру (Tab, Enter, Esc для закрытия)
- Контраст карточек соответствует WCAG 2.1 AA
- Focus-состояние дублирует hover
