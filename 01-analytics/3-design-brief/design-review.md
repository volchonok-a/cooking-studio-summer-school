# Design Review: Чек-лист ревью макетов

## 1. Описание
Чек-лист для проверки макетов дизайнера на соответствие функциональным (FR) и нефункциональным (NFR) требованиям. Используется на этапе приёмки дизайна до передачи в разработку.

**Цель:** убедиться, что все экраны, состояния и UX-паттерны покрыты макетами и соответствуют требованиям.

## 8. Критерии приёмки дизайна

- [ ] Все 10 экранов (SCR-001...SCR-010) спроектированы
- [ ] Предусмотрены hover и focus-состояния для всех интерактивных элементов
- [ ] Есть empty states, error states, loading states
- [ ] Есть состояния блокировки и офлайн-режима
- [ ] Путь бронирования укладывается в ≤ 3 экрана
- [ ] Контраст соответствует WCAG 2.1 AA
- [ ] Все тексты вынесены отдельно (под i18n)
- [ ] Дизайн-система / UI-кит согласован с разработкой
